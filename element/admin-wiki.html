Quick Start

  1. Drop SLElemental.jar into /plugins/
  2. Start server — all config files auto-generate
  3. Edit configs, then /element admin reload
MMOItems (required) MythicLib (required) MythicMobs PlaceholderAPI Vault

Supports Minecraft 1.20 — 1.21.1

config.yml — Main Settings

Weapon Effectiveness

effectiveness:
  strong-multiplier: 1.25   # +25% damage when strong
  weak-multiplier: 0.75     # -25% damage when weak

Visual Effects Toggles

visual:
  particles: true            # Particle burst on hit
  sounds: true               # Element-specific hit sounds
  effects: true              # On-hit potion effects
  head-indicator: true       # Element name above target
  weapon-aura: true          # Ambient particles on weapon
  effectiveness-text: true   # Floating damage text

Visual Timing & Aura Tuning

visual:
  # --- Timing (ticks, 20 ticks = 1 second) ---
  head-indicator-duration: 40   # How long indicator stays (40 = 2s)
  hologram-duration: 30         # How long effectiveness text stays (30 = 1.5s)
  hologram-rise-height: 1.2     # Height above target for floating text

  # --- Weapon Aura Tuning ---
  aura-tick-interval: 10        # How often aura spawns (10 = every 0.5s)
  aura-particle-ratio: 0.25     # 25% of on-hit particle count
  aura-spread-ratio: 0.30       # 30% of on-hit spread
  aura-speed-ratio: 0.50        # 50% of on-hit speed

Armor Element System

armor-element:
  enabled: true
  mode: dominant             # Majority element across 4 pieces

  damage:
    strong-reduction: 0.80   # Armor resists = 20% less damage
    weak-increase: 1.20      # Armor weak = 20% more damage
    same-element: 0.90       # Same element = 10% resistance

  durability:
    enabled: true
    strong-multiplier: 0.70  # 30% less durability loss
    weak-multiplier: 1.50    # 50% more durability loss
    same-element: 0.85       # 15% less durability loss

  pvp:
    damage-cap: 1.15          # Max +15% in PvP
    damage-floor: 0.85        # Max -15% in PvP
    durability-scale: 0.60    # 60% of PvE effect

  set-bonus:
    enabled: true
    full-set-reduction: 0.10  # Extra 10% DR for 4 matching
    full-set-aura: true
Tip: All values are hot-reloadable. Edit, save, then /element admin reload. No restart needed.

Item Element Assignment (Click-to-Pick)

/element admin → Set Item Element → Click MMOItem in inventory → Toggle Elements (max 2)
  1. Click "Set Item Element" in Admin Panel
  2. A 1-row GUI opens — click any MMOItem in your inventory below
  3. The item is taken and the Element Toggle Picker opens
  4. Toggle elements on/off (up to 2: primary + secondary)
    • ✔ SELECTED — element icon with colored name
    • ○ Not selected — gray dye
  5. Click "✔ Confirm & Apply" to save — item returned to inventory
Safety: Closing GUI with ESC or clicking Cancel always returns the item untouched. If inventory is full, item drops at your feet.

elements.yml — Element Definitions

Each element needs a unique ID and the following properties:

elements:
  fire:
    display-name: "Fire"
    color: "#FF6B35"               # Hex color used everywhere
    strong-against: [ice, poison]    # Elements this beats
    weak-to: [water, earth]          # Elements that beat this
    gui-material: RED_DYE            # Material in GUI menus
    description:
      - "<color:#FF6B35>Burns enemies</color>"
    on-hit-effect:
      name: "Burn"
      fire-ticks: 60                 # Set on fire (ticks)
      potion-effects:
        - { type: SLOWNESS, duration: 40, amplifier: 0 }
    particle:
      type: FLAME     count: 25     spread: 0.4     speed: 0.05
    sound:
      on-hit: ENTITY_BLAZE_SHOOT
      on-strong: ENTITY_GENERIC_EXPLODE
      on-weak: BLOCK_FIRE_EXTINGUISH
Adding custom elements: Copy any element block, change the ID and properties, and reload. No code changes needed.

mob-elements.yml — Mob Assignments

# MythicMobs — use internal mob ID
mythicmobs:
  FireDragon: fire,draconic      # Dual-type!
  IceGolem: ice

# Vanilla — use EntityType names
vanilla-mobs:
  BLAZE: fire
  ENDER_DRAGON: draconic
  ZOMBIE: dark

Can also assign via GUI: /element admin → Mob Assignment

type-chart.yml — Override Matchups

Fine-tune specific matchups without touching elements.yml:

overrides:
  fire.ice: 1.50       # Fire does 1.5x vs Ice (instead of 1.25x)
  normal.ghost: 0.0    # Complete immunity

Combat Flow

Weapon vs Mob (Offensive)

Player (Fire sword) → Ice Mob = base_damage x 1.25

Triggers: on-hit effect (Burn), particle burst, sound, floating text "Strong! +25%", element name above target.

Boss vs Player Armor (Defensive)

Fire Boss → Player (Water armor) = base_damage x 0.80

Water is strong against Fire, so the player takes 20% less damage. Each Water armor piece also loses 30% less durability.

Armor Element System

The system reads each of the 4 armor pieces and calculates a dominant element based on majority.

LayerHow it WorksValues
Damage ModifierDominant element vs attackerStrong: 0.80 / Weak: 1.20 / Same: 0.90
Durability DrainEach piece individuallyStrong: x0.70 / Weak: x1.50 / Same: x0.85
Set BonusAll 4 pieces matchExtra 10% damage reduction
PvP CapsPlayer vs Player limitsDamage: 0.85 — 1.15 / Durability: 60%

Dual-Type System

When both attacker and defender have 2 elements, all 4 pairings are calculated and averaged:

Sword (Fire + Ice) → Boss (Water + Earth)
PairMatchupMultiplier
Fire vs WaterWeak0.75
Fire vs EarthWeak0.75
Ice vs WaterStrong1.25
Ice vs EarthNeutral1.00
Average0.9375

MMOItems Integration

SLElemental registers a custom stat "SL Element" directly into MMOItems’ item editor.

Method 1: Via /mi browse (SL Element Stat)

/mi browse → Select type (SWORD, BOW...) → Select item → Find "SL Element" (Blaze Powder)
  1. Navigate to the ELEMENTAL page in the item editor
  2. Find "SL Element" stat (Blaze Powder icon)
  3. Left-Click → type element ID in chat (e.g. fire)
  4. Right-Click → remove element
Note: The "SL Element" stat is separate from MMOItems' built-in "Elements" (Fire Damage, Water Defense, etc). They do not conflict.

Method 2: Via Admin Panel GUI

Use /element admin → Set Item Element for a visual click-picker with toggle support (up to 2 elements). See Item Element Assignment above.

Item Lore Format

Customize how the element appears on item lore in config.yml:

integration:
  mmoitems:
    lore-format: "&7Element: {element_name}"   # Placeholders: {element_name}, {element_color}

Balance Testing Workflow

  1. Edit values in config.yml
  2. Run /element admin reload
  3. Test in-game
  4. Repeat until balanced
No restart needed. All multipliers, toggles, damage values, durability multipliers, and PvP caps are reloaded instantly.
