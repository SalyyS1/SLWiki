OmniTooltips

Multi-page tooltip system for MMOItems. Each item can have multiple tooltip pages with different stats, lore, abilities, and lore-format layouts.

How It Works Setup Page YAML Config Per-Page Fields Per-Page Lore Format Full Example Templates & Type Defaults Commands

How It Works

You define page-1:, page-2:, etc. sections inside your MMOItems item YAML. Each page lists which stats to display. Stats from all pages are merged into base: automatically — they always work regardless of which page is shown.

Players cycle pages with Shift + Right-Click in inventory. Items always start on page 1 when given or picked up.

Setup

  1. Drop OmniTooltips.jar into plugins/
  2. Requires Paper 1.21+, MMOItems 6.10+, MythicLib 1.7+
  3. Start server — config generates automatically
  4. Add page sections to your item YAML files in plugins/MMOItems/item/

Page YAML Config

Add page-N: sections to any item in plugins/MMOItems/item/SWORD.yml (or any type).

MY_SWORD:
  base:
    material: GOLDEN_SWORD
    name: "&6Flame Blade"
    tier: LEGENDARY

  page-1:
    attack-damage:
      base: 85
    critical-strike-chance:
      base: 25
    lore:
      - "&7A blade of ancient fire."

  page-2:
    ability:
      ability1:
        type: SPARKLE
        mode: RIGHT_CLICK
    lore:
      - "&7Skills and abilities."
Each stat should only appear in ONE page. OmniTooltips merges all page stats into base so MMOItems can load them, but each stat only displays on its assigned page.

Per-Page Lore Format

Each page can use a different lore-format file. This controls the layout order of stats, lore, tier, etc. in the tooltip.

Creating a custom format

Create a new .yml file in plugins/MMOItems/language/lore-format/:

# plugins/MMOItems/language/lore-format/minimal.yml
lore-format:
  - '#item-name#'
  - ''
  - '#lore#'
  - ''
  - '#page-indicator#'

Using it in a page

page-2:
  lore-format: minimal
  ability:
    ability1:
      type: SPARKLE
      mode: RIGHT_CLICK

The format name is the filename without .yml. The default format (lore-format.yml) is named default.

Full Example

A 3-page sword with different formats, toggles, and name overrides per page:

# plugins/MMOItems/item/SWORD.yml
DAWNBREAKER_BLADE:
  base:
    material: GOLDEN_SWORD
    name: "&6&lDawnbreaker Blade"
    tier: LEGENDARY
    custom-model-data: 7001
    unbreakable: true
    hide-enchants: true
    enchants:
      sharpness: 5
      fire-aspect: 2
    required-level: 30

  # Page 1 — Combat stats, default format, everything visible
  page-1:
    lore-format: default
    show-tier: true
    show-enchants: true
    show-required-level: true
    attack-damage:
      base: 85
      scale: 3.5
    critical-strike-chance:
      base: 25
    critical-strike-power:
      base: 150
    max-health:
      base: 40
    lore:
      - "&7Forged in the first light of dawn."

  # Page 2 — Gems, different format, hide tier and enchants
  page-2:
    lore-format: minimal
    show-tier: false
    show-enchants: false
    show-upgrade: false
    show-required-level: false
    item-name-override: "&6&lDawnbreaker Blade &7(Gems)"
    attack-speed:
      base: 1.4
    armor:
      base: 5
    gem-sockets:
      - "Offensive"
      - "Offensive"
      - "Defensive"
    lore:
      - "&7Socket loadout page."

  # Page 3 — Abilities only, clean layout
  page-3:
    lore-format: default
    show-tier: false
    show-enchants: false
    show-set-bonus: false
    show-upgrade: false
    show-required-level: false
    show-item-level: false
    show-durability: false
    show-displayed-type: false
    item-name-override: "&d&lDawnbreaker Blade &7(Skills)"
    ability:
      ability1:
        type: SPARKLE
        mode: RIGHT_CLICK
        cooldown: 8
        mana: 25
        damage: 120
      ability2:
        type: FIRE_METEOR
        mode: SHIFT_RIGHT_CLICK
        cooldown: 15
        mana: 60
        damage: 250
    lore:
      - "&7Active abilities."

Templates & Type Defaults

Instead of defining pages per-item, you can create templates in config.yml and assign them to item types.

Template definition

# In config.yml
templates:
  combat:
    name: "Combat"
    lore-format: default
    stats:
      - ATTACK_DAMAGE
      - ATTACK_SPEED
      - CRITICAL_STRIKE_CHANCE
    skills: ["*"]
    lore: true

  full-weapon:
    pages:
      - combat
      - story

Type defaults

All items of a given type will use the specified template automatically:

type-defaults:
  SWORD: full-weapon
  BOW: full-weapon
  ARMOR: full-armor

Items with an explicit TOOLTIP_PAGES stat override their type default.
