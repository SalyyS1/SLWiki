Admin Guide

Feature Setup & Mechanics

Setup guide for each feature — drag-drop, protection stone, tier shop GUI, extraction, fusion, grindstone, enchant table, soul tracking, trak system.

📦 Drag & Drop Enchanting

The primary mechanic for applying enchants to MMOItems. Players drag enchant books onto items in their inventory.

Internal Processing Flow
# 1. Player drags enchant book → drops onto MMOItems item
# 2. Plugin checks:
#    ✓ applicable-to: is the item type valid?
#    ✓ conflicts: does it conflict with existing enchants?
#    ✓ max-enchants: has the limit been reached? (base + extra slots)
#    ✓ allow-stacking: does this enchant already exist?
# 3. Roll success-rate (from book + Lucky Dust bonus)
# 4. Success → Apply enchant + stats + lore
# 5. Failure → Roll failure-rate →
#    Hit + has Protection Stone → consume protection
#    Hit + no protection → DESTROY item
#    Miss → only lose the book
⚙️

Config: settings/application.yml → drag-drop section
Toggle: drag-drop.enabled: true/false
Important: Slot limit = config.yml → max-enchants-per-item + NBT SLENCHANT_EXTRA_SLOTS (from Slot Increaser)

🛡️ Protection Stone

An item that protects gear from being destroyed when enchant application fails. Each item can hold 1 protection.

How to Set Up
# Create Protection Stone for player:
/sle give <player> protection 1

# Player drags Protection Stone onto item → lore shows "Protected: 1x"
# When enchant application fails + destroy roll hits:
#   → Protection consumed, item is saved
#   → Message: protection-saved (from messages.yml)

# Default material: TOTEM_OF_UNDYING
# NBT tag: SLENCHANT_PROTECTED = 1

🏪 Tier Shop GUI

GUI shop where players buy Enchant Orbs by rarity tier. Fully customizable: slots, materials, prices, lore.

Setup Tier Shop
# 1. Config: settings/tier-shop.yml
# 2. Open GUI: /sle shop
# 3. Flow:
#    Player clicks tier button → currency deducted → receives Orb item
#    Player right-clicks Orb → reveals random enchant book
#    Book has RANDOMIZED success/failure rate:
#      success = base ±15%
#      failure = 0 to 2× base

# Currency: EXP (default), VAULT, POINTS (PlayerPoints)
# Change: tier-shop.yml → currency: VAULT
!

Customize GUI: See Configuration → tier-shop.yml to customize slots, materials, prices, lore for each tier. DeluxeMenus-like style.

🔬 Extraction System

Remove enchants from items and convert them back into enchant books. May require an Extraction Scroll.

Setup & Usage
# Config: settings/extraction.yml
# Extract commands:
/sle extract                      # Remove random enchant
/sle extract <enchant_id>          # Remove specific enchant

# Create Extraction Scroll for player:
/sle give <player> scroll 1

# If require-scroll: true → player needs scroll in off-hand
# success-rate: 100 → always succeeds (lower for risk)
# level-penalty: 0 → keeps level (set 1 to reduce by 1)

🔀 Fusion System

Combine multiple enchant books to create a more powerful enchant. Requires recipes defined in config.

Create Fusion Recipe
# Config: settings/fusion.yml
# Open GUI: /sle fusion

# Example recipe: 3× Strength I → 1× Strength II
recipes:
  strength_2:
    result:
      enchant: strength
      level: 2
    ingredients:
      - enchant: strength
        level: 1
        amount: 3

🪨 Grindstone Integration

Vanilla Grindstone automatically integrates with SLE enchants. No config needed.

Behavior
# Grindstone works automatically:
# 1. Item with both vanilla + custom enchants → removes all
# 2. Item with only custom enchants (no vanilla) → still removable
#    Plugin auto-creates result for grindstone
# 3. Lore cleaned immediately in preview (no interval scan)
# 4. Stats recalculated instantly when taking item from result slot

📖 Enchant Table Integration

Custom enchants appear when using the vanilla enchanting table. Controlled via whitelist/blacklist and weight.

Setup
# Config: settings/enchant-table.yml
# Enable: enabled: true
# Blacklist mode (default): all enchants appear except those listed
# Whitelist mode: only enchants in list appear

# Adjust weight for specific enchants:
enchants:
  telekinesis:
    weight: 3.0  # 3× more likely than normal
  frenzy:
    weight: 0.1  # Very rare from enchanting table

💀 Soul System

Enchants with soul-cost consume souls from the item each time they activate. Players collect souls using Soul Stone + Soul Booster.

Setting Up Souls for Enchants
# In enchant YAML:
soul-cost: 5                       # Costs 5 souls per activation

# Create items for player:
/sle give <player> soul_pvp 1     # Soul Stone (enables tracking)
/sle give <player> soul_booster_pvp 1  # +Souls
/sle give <player> soul_booster_pve 1  # +Souls

# Flow:
# 1. Drag Soul Stone onto item → enables soul tracking
# 2. Drag Soul Booster onto item → +souls
# 3. Enchant activates → checks soul-cost → enough → deducts souls
#    Not enough souls → enchant DOES NOT activate

📈 Trak System

Counts blocks mined and mobs killed with an item. Displayed on item lore.

Setup Trak
# Create Trak Stone for player:
/sle give <player> trak_block 1   # Block tracker
/sle give <player> trak_kill 1    # Kill tracker
/sle give <player> trak_mob 1     # Mob kill tracker

# Drag onto item → starts tracking
# Lore auto-shows counter: "⛏ Blocks: 1,234" or "💀 Kills: 567"

# PAPI Placeholders:
# %slenchant_trak_blocks% — total blocks mined
# %slenchant_trak_kills% — total kills
# %slenchant_trak_mob_kills% — total mob kills

⬆️ Slot Increaser

Increases the maximum number of enchants allowed on a single item.

Setup
# Create Slot Increaser:
/sle give <player> slot_increaser 1

# Drag onto item → +1 max enchant slot
# Extra slot limit: application.yml → extra-slot-limit: 5
# E.g.: base=6, extra=3 → that item holds max 9 enchants

# NBT stored: SLENCHANT_EXTRA_SLOTS (integer)
