Admin Guide

Configuration Reference

All configuration files in plugins/SLMMOEnchant/ — complete guide for every field, default values, and usage.

⚙️ config.yml — General Settings

The main configuration file that controls general plugin behavior.

config.yml
version: 1                         # Do not edit (auto migration)
debug: false                        # Enable detailed logging (dev only)
book-material: ENCHANTED_BOOK       # Material used for enchant books
max-enchants-per-item: 6            # Maximum enchants per item (base)
allow-stacking: false              # Allow duplicate enchants?
inventory-interval: 200            # Ticks between inventory scans (200=10s)
lore-spacing: 0                    # Gap between enchant lore lines
!

max-enchants-per-item is the BASE value. If using Slot Increaser (Nether Star), items gain extra slots on top of this base. E.g.: base=6, extra=2 → max=8.

🔨 settings/application.yml — Drag-Drop & Rates

Controls the enchant application mechanic: success/failure rates, upgrade behavior, unique activation, extra slot limit.

settings/application.ymlFull reference
drag-drop:
  enabled: true                    # Enable/disable drag-drop enchanting
  base-success-rate: 85            # Default success % rate
  base-failure-rate: 10            # % chance to destroy item on failure
  allow-upgrade-on-apply: true     # Same enchant+level → auto upgrade
  unique-activation: true         # Keep only highest level for duplicate enchants

extra-slot-limit: 5               # Max extra slots from Slot Increaser (0=unlimited)

dust:
  enabled: true                    # Enable Lucky Dust
  bonus-per-dust: 1               # +1% success per dust
  max-bonus: 15                   # Max cap +15%
i

Enchanting flow:
1. Success rate = success-rate on book (from enchant config or /sle give random)
2. Failure → roll failure-rate → hit = destroy item (unless Protection Stone)
3. Lucky Dust increases success rate (adds % directly)

🏪 settings/tier-shop.yml — GUI Shop Setup

Configure the Enchant Orb purchase GUI. DeluxeMenus-like style — customize slot, material, lore, price per tier.

settings/tier-shop.ymlGUI customization
enabled: true
currency: EXP                      # EXP, VAULT, POINTS (PlayerPoints)
orb-material: MAGMA_CREAM          # Material for Orb item

gui:
  title: "&5&l✦ Enchant Shop"       # GUI title
  rows: 3                           # Number of rows (1-6)
  background:                        # Background fill border
    slots: [0,1,2,...,26]           # Slot numbers (0-indexed)
    material: BLACK_STAINED_GLASS_PANE
    name: " "
  info:                              # Center info item
    slot: 13
    material: NETHER_STAR
    name: "&e&lEnchant Shop"
    lore: ["&7Buy Enchant Orbs..."]

tiers:                               # Each tier = 1 button in GUI
  COMMON:
    slot: 10                        # Position in GUI
    display-name: "&f✦ Common Orb"
    material: WHITE_DYE
    price: 10                       # Price (currency unit)
    description:                     # Display lore
      - "&7Contains a random &fCommon &7enchant"
      - "&7Price: &a{price} Levels"   # {price} = auto fill
      - "&eClick to purchase!"
  LEGENDARY:
    slot: 16
    display-name: "&6✦ Legendary Orb"
    material: ORANGE_DYE
    price: 75
!

Open GUI: /sle shop or /sle tiershop
Flow: Buy Orb → Right-click to reveal → Receive random enchant book of that tier, with randomized success/failure rate (base ±15%, failure 0-2×base).

📖 settings/enchant-table.yml — Enchant Table Integration

Configure how custom enchants appear when using the vanilla enchanting table.

settings/enchant-table.yml
enabled: true

enchant-list:
  mode: false                      # true=WHITELIST, false=BLACKLIST
  list: []                         # Enchant IDs

# Override weight/level per enchant
enchants:
  aegis:
    weight: 2.0                   # 2× appearance rate
    min-level: 1
    max-level: 2                  # Cap level from enchanting table
  frenzy:
    weight: 0.5                   # 0.5× rate (rarer)
i

Weight controls relative appearance rate. Enchants not in the list use weight=1.0 by default. max-level is capped by the max-level in the enchant YAML file.

🔬 settings/extraction.yml — Enchant Extraction

settings/extraction.yml
enabled: true
require-scroll: true              # Requires Extraction Scroll to use
scroll-material: PAPER            # Material for scroll item
success-rate: 100                 # % success rate for extraction
level-penalty: 0                  # Reduce enchant level after extraction (0=keep)
i

Command: /sle extract (random enchant) or /sle extract <enchant_id> (specific).
If require-scroll: true, player must hold Extraction Scroll in off-hand. Create scroll: /sle give <player> scroll <amount>.

🔀 settings/fusion.yml — Fusion Recipes

Define recipes for combining enchant books into more powerful enchants.

settings/fusion.yml
enabled: true
gui:
  title: "&d&l✦ Enchant Fusion"
  rows: 3
recipes:
  strength_2:                        # Recipe ID (custom name)
    result:
      enchant: strength              # Output enchant
      level: 2                      # Output level
    ingredients:                      # Required enchant books
      - enchant: strength
        level: 1
        amount: 3                    # Needs 3 Strength I books

📜 settings/vanilla-enchants.yml — Override Vanilla

Override display name, description, and rarity for vanilla Minecraft enchants in SLE lore block.

settings/vanilla-enchants.yml
enabled: true                      # Show vanilla enchants in SLE lore
show-in-lore: true                 # Display vanilla enchant info
allow-drag-drop: true              # Allow dragging vanilla books onto MMOItems
enchants:
  sharpness:
    display-name: "&fSharpness"
    description: "&7Increases melee damage"
    rarity: COMMON
  protection:
    display-name: "&fProtection"
    description: "&7Reduces damage taken"
    rarity: COMMON
