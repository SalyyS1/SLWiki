FAQ

Does the plugin support Folia?

Yes. Uses FoliaLib — compatible with both Paper and Folia.

Create new enchants without coding?

Absolutely! Add a .yml file to enchants/ then /sle reload.

Can enchants combine multiple types?

Yes! For example, Divine has stats + ability simultaneously. Spirit has stats + heal ability.

What if I apply the same book at the same level?

Automatically upgrades to level+1. Can be disabled in application.yml.

How does Protection Stone work?

Drag onto item — protects once. On enchant failure + destroy roll — consumes protection instead of destroying item. Supports stacking charges.

What does the Tier Shop sell?

Buy Enchant Orbs by tier — right-click to open — receive random enchant book with randomized success/destroy rates.

Does Grindstone remove custom enchants?

Yes! Grindstone removes both vanilla and custom enchants, lore is cleaned immediately.

Where do enchant lore lines appear on items?

By default at the top. Use #omnienchant# placeholder in your MMOItems lore format to control exact position. See the Lore Format page.

What changed from SLMMOEnchant to OmniEnchant?

Plugin was rebranded in v3.0.0. All legacy placeholders (#slenchant#, #ominienchant#) and permission nodes (slenchant.*) still work. Existing item data is preserved.
