Admin Guide

Enchant YAML Reference

Complete guide to creating & configuring custom enchants — stats, abilities, mechanics, triggers, formulas, placeholders, and conflict groups.

📁 File Structure

Each enchant is a separate YAML file in the plugins/SLMMOEnchant/enchants/ folder. File name = enchant ID (e.g., aegis.yml → ID aegis).

Directory Structure
plugins/SLMMOEnchant/
├── config.yml                # General settings
├── enchant.yml               # Display & rarity config
├── settings/
│   ├── application.yml       # Drag-drop, success/failure rate
│   ├── display.yml           # Rarity colors, display mode
│   ├── dust.yml              # Lucky Dust config
│   ├── enchant-browser.yml   # Enchant Browser GUI
│   ├── enchant-table.yml     # Custom Enchant Table
│   ├── extraction.yml        # Extraction Scroll config
│   ├── fusion.yml            # Fusion recipes
│   ├── tier-shop.yml         # Tier Shop GUI
│   └── vanilla-enchants.yml  # Vanilla enchant overrides
└── enchants/
    ├── aegis.yml             # Each file = 1 enchant
    ├── divine.yml
    ├── frenzy.yml
    └── ...

⚙️ Base Fields (Required)

Every enchant YAML file must have these base fields:

enchants/my_enchant.ymlFull example
name: "&6Aegis"                   # Display name (color codes)
lore:                                # Enchant description, {KEY} = placeholder
  - "&8Balanced protection granting &a{HP_PCT}% &8max health"
  - "&8and &a{DEF_PCT}% &8defense."
max-level: 5                       # Maximum enchant level
rarity: EPIC                       # COMMON, UNCOMMON, RARE, EPIC, LEGENDARY
success-rate: 70                   # % base success when applying (optional)
failure-rate: 5                    # % chance to destroy item on failure (optional)
applicable-to: [CHESTPLATE, HELMET] # Allowed item types
conflicts: [umbra]                # Conflicts with other enchants (optional)
!

applicable-to supports: SWORD, AXE, BOW, CROSSBOW, PICKAXE, SHOVEL, HOE, HELMET, CHESTPLATE, LEGGINGS, BOOTS, SHIELD, ARMOR (= all armor), WEAPON (= all weapons).

📊 Stats — Permanent & Conditional

Add/modify MythicLib stats on items. Supports both permanent (always active) and conditional (trigger-based) stats.

Permanent Stats (Always Active)

Example: Warlord (multi-stat)
stats:
  attack-damage:                     # MythicLib stat ID
    formula: "1.5+({level}-1)*1"      # Value scaled by enchant level
    key: AD                           # Placeholder: {AD} in lore
  physical-damage:
    formula: "12+({level}-1)*8"
    key: PD
  critical-strike-power:
    formula: "10+({level}-1)*10"
    key: CSP
  critical-strike-chance:
    formula: "5+({level}-1)*2.5"
    key: CSC

Relative Stats (Percentage-Based)

Add type: RELATIVE to make the stat a percentage modifier instead of flat value.

Example: Titan (relative %)
stats:
  attack-damage:
    formula: "2+({level}-1)*1"
    key: AD_PCT
    type: RELATIVE                   # Percentage-based modifier
  max-health:
    formula: "3+({level}-1)*1.5"
    key: HP_PCT
    type: RELATIVE

Conditional Stats (Trigger-Based)

Add mode to make a stat only apply temporarily when a trigger occurs (e.g., on attack → buff crit for 5s).

Example: Buff Crit on AttackConditional
stats:
  critical-strike-chance:
    formula: "95+({level}-1)*1"   # Stat value
    key: CSC                       # Placeholder: {CSC}
    mode: ATTACK                   # MythicLib TriggerType ⚡
    chance:                         # Activation chance (%)
      formula: "5+({level}-1)*5"
      key: CHANCE                 # Placeholder: {CHANCE}
    duration:                       # Buff duration (ticks)
      formula: "100"             # 100 ticks = 5s
      key: DUR                   # Placeholder: {DUR}
    cooldown:                       # Cooldown between activations
      formula: "200"             # 200 ticks = 10s
      key: CD                    # Placeholder: {CD}
⚡

mode must use actual MythicLib TriggerType:

ATTACKWhen attacking entityWHEN_HITWhen hit by entity
DAMAGEDWhen receiving any damageDAMAGED_BY_ENTITYWhen damaged by entity
KILLWhen killing entitySHOOT_BOWWhen shooting bow
RIGHT_CLICKRight clickLEFT_CLICKLeft click
SHIFT_RIGHT_CLICKShift + right clickSNEAKWhen sneaking
TIMERPeriodicLOGINWhen joining server
i

chance, duration, cooldown support 2 formats:
• Simple: chance: "50" — formula only
• Sub-object: chance: { formula: "5+{level}*5", key: CHANCE } — formula + placeholder key for lore

⚔️ Abilities — MythicLib Skills

Attach MythicLib skills to enchants. Each ability has a type (skill ID), mode (trigger), and modifiers (damage, cooldown, chance...).

Example: Divine Heal Ability
ability:
  smite_heal:                       # Ability ID (custom name)
    type: HEAL                        # MythicLib SkillHandler ID ⚡
    mode: ATTACK                      # MythicLib TriggerType
    heal:                             # Modifier: heal amount
      formula: "2+({level}-1)*1"
      key: HEAL
    chance:                           # Activation chance
      formula: "20"
Example: Spirit Heal (Defensive)
ability:
  spirit_heal:
    type: HEAL
    mode: DAMAGED_BY_ENTITY           # Triggers when attacked
    heal:
      formula: "5+({level}-1)*2"
      key: HEAL
    chance:
      formula: "5+({level}-1)*2"
      key: CHANCE
⚡

type is the MythicLib SkillHandler ID. Some common ones:
FIREBALL, SPARKLE, EARTH_SPIKE, FIRE_METEOR, HEAL, ARCANE_HAIL, FREEZING_CURSE, POWER_MARK, SHULKER_MISSILE...
See full list: MythicLib Skill Wiki

⚡ Conflict System

Enchants can be set to conflict with each other, preventing both from being applied to the same item simultaneously.

Conflict Examples
# Divine ↔ Umbra (Holy pair)
# divine.yml:
conflicts: [umbra]

# Reaper ↔ Ruin (Cursed pair)
# reaper.yml:
conflicts: [ruin]

# Bulwark ↔ Vitality (Titan pair)
# bulwark.yml:
conflicts: [vitality]

# Juggernaut ↔ Lethality (Heavy pair)
# juggernaut.yml:
conflicts: [lethality]
i

Conflicts are bidirectional — if Divine conflicts with Umbra, then Umbra also conflicts with Divine. You should define conflicts on both sides for clarity.

🎯 Triggers — Temporary Buffs

Temporary stat buffs on events. Unlike conditional stats, triggers can apply multiple effects at once for a set duration.

Example: Regeneration (heal when hit)
trigger:
  condition: ON_DEFEND              # Alias → resolves to WHEN_HIT
  threshold: 0                      # Min damage to trigger
  duration: 100                     # Buff duration (ticks), 100 = 5s
  cooldown: 1000                    # Cooldown between triggers (ticks)
  effects:
    health-regeneration:             # MythicLib stat ID
      formula: "1+({level}-1)*0.5"
      key: REGEN                   # Placeholder: {REGEN}

📐 Formula Engine

All numeric values support formulas with {level} (enchant level) and {item_level} (MMOItems level) variables.

Formula Syntax
# Basic
"10"                    # Fixed value
"5+({level}-1)*3"       # Lv1=5, Lv2=8, Lv3=11
"10-({level}-1)*0.5"    # Decreasing (cooldown)

# Supported operators: + - * / ^ ()
"({level}^2)*2"         # Exponential scaling
"100/{level}"           # Inverse scaling

# Negative values (trade-off enchants)
"-0.5+({level}-1)*-0.2" # Frenzy: reduces defense per level
"-0.001+({level}-1)*-0.0005" # Bulwark: reduces speed

🏷️ Lore Placeholders

Use {KEY} in lore: to display dynamic values calculated from formulas.

PlaceholderSourceDescription
{KEY}stat/abilityFormula value at current level
{DUR}conditional stat/triggerDuration (seconds, auto ÷20 from ticks)
{CD}conditional stat/triggerCooldown (seconds)
{CHANCE}conditional stat/abilityActivation chance (%)
{CHANCE_ENCHANT}base fieldSuccess rate from success-rate
{MULT}ability/mechanicMultiplier value
{value}statsFirst stat value (legacy)
{STATID_DUR}conditional statDuration for specific stat ID
{STATID_CD}conditional statCooldown for specific stat ID
i

If a sub-object has a key, the placeholder uses that key: chance: { formula: "50", key: RATE } → {RATE}. Without a key, use the generic {CHANCE}.

📋 Full Example — Complete Enchant

enchants/frenzy.ymlTrade-off enchant: offense at the cost of defense
# Frenzy — High damage at the cost of defense
name: "&cFrenzy"
lore:
  - "&8Increases attack damage by &a{AD_PCT}% &8and critical"
  - "&8chance by &a{CSC}%&8, but reduces defense by &a{DEF}&8."
max-level: 4
rarity: EPIC
applicable-to: [SWORD]

stats:
  attack-damage:
    formula: "4+({level}-1)*1.5"
    key: AD_PCT
    type: RELATIVE                   # Percentage-based
  critical-strike-chance:
    formula: "6+({level}-1)*1.5"
    key: CSC
  defense:
    formula: "-0.5+({level}-1)*-0.2"  # Negative = reduces defense
    key: DEF

applicable-to: [HELMET, CHESTPLATE]
