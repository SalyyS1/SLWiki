OmniTotalStats — Overview

OmniTotalStats displays real-time total stat values on item lore. It reads from MythicLib's StatMap which already aggregates everything: vanilla base values, MMOCore class bonuses, MMOInventory equipment, enchantments, buffs — all sources combined.

Each stat you define in the config automatically becomes an MMOItems stat that you can:

  • Enable/disable per item via the MMOItems GUI editor
  • Position in item lore using #your-key# tokens
  • Customize the display format in stats.yml
How it works When a player holds or equips an item, OmniTotalStats calculates the total value from the player's entire StatMap and displays it live on the item's lore. The value updates when equipment changes.

Installation

  1. Requirements: Paper 1.21+, MMOItems, MythicLib, MMOCore (optional), MMOInventory (optional).
  2. Drop OmniTotalStats-1.0.jar into your plugins/ folder.
  3. Restart the server. The plugin generates plugins/OmniTotalStats/config.yml.
  4. Configure stats in config.yml, then set up lore tokens in MMOItems files.

Quick Start — 3-Step Setup

Let's add "Total PvE Damage" to show on swords. Here are the 3 files you need to touch:

Step 1: Define the stat in config.yml

# plugins/OmniTotalStats/config.yml
stats:
  melee-pve:                     # ← This is the stat ID
    type: damage
    damage-type: melee
    context: pve
    show-enable: true
    item-types: [sword, dagger]
    material: DIAMOND_SWORD

Step 2: Add the lore token in lore-format.yml

# plugins/MMOItems/language/lore-format.yml
lore-format:
  - '{bar}'
  - '{attack_damage}'
  - '#melee-pve#'        # ← uses the stat ID wrapped in #
  - '{bar}'
The token format is #stat-id# The ID inside the # markers must exactly match the key you defined in config.yml. In this case: melee-pve in config → #melee-pve# in lore-format.

Step 3: Define the display text in stats.yml

# plugins/MMOItems/language/stats.yml
melee-pve: "&7Total PvE Dmg: &f{value}"

The {value} placeholder is replaced by the calculated total at runtime. Use any color codes you want: &a, &e, 7FF55 (hex), etc.

Result Your sword will now show something like:
Total PvE Dmg: 42.5
The value updates live based on the player's full equipment + bonuses.

Stat Types Reference

TypeDescriptionKey Options
stat Reads a single stat directly from the player's StatMap stat: MAX_HEALTH
damage Calculates total weapon damage: ATK × (1 + bonuses/100) damage-type: + context:
element Elemental damage bonus: ELE_DMG × (1 + ELE_DMG%/100) element: FIRE
damage-element Combined: weapon damage + elemental bonus damage-type: + element:
formula Custom math expression with stat placeholders formula: "..." + decimals:

Lore Format Setup (lore-format.yml)

The lore-format.yml file in plugins/MMOItems/language/ controls the order and layout of stats on item lore. To add your OmniTotalStats entries, place #your-stat-id# tokens wherever you want them to appear. 

# plugins/MMOItems/language/lore-format.yml
lore-format:
  - '{bar}'
  # Vanilla stats
  - '{attack_damage}'
  - '{attack_speed}'
  - '{critical_strike_chance}'
  - '{bar}'

  # === OmniTotalStats entries ===
  - '#melee-pve#'           # Total melee PvE damage
  - '#fire-damage#'         # Total fire elemental damage
  - '#melee-fire-pve#'      # Total melee + fire combined
  - '#total-max-health#'    # Total max HP
  - '#total-defense#'       # Total defense
  # You can add ANY stat ID from config.yml here

  - '{bar}'
  - '{can-identify}'
  - '{can-deconstruct}'
Only shows when enabled A #stat-id# token only shows on items where that stat is enabled in the MMOItems GUI editor. If the stat is disabled (or the item type doesn't match item-types), the line is silently hidden.

Stats.yml — Display Format

The stats.yml file in plugins/MMOItems/language/ controls how each stat's value is displayed. Add one entry per stat ID, using {value} as the placeholder: 

# plugins/MMOItems/language/stats.yml

# Damage types
melee-pve: "&7Total PvE Dmg: &f{value}"
melee-pvp: "&7Total PvP Dmg: &f{value}"
magic-pve: "&7Total Magic Dmg: &f{value}"

# Elemental
fire-damage: "&c Fire Damage: &f{value}"
ice-damage: "&b Ice Damage: &f{value}"
earth-damage: "&2 Earth Damage: &f{value}"

# Combined (melee + element)
melee-fire-pve: "&c Total Fire Hit: &f{value}"

# Defensive stats
total-max-health: "&c Total HP: &f{value}"
total-defense: "&e Total Defense: &f{value}"
total-armor: "&8 Total Armor: &f{value}"

# Resources
total-max-mana: "&9 Total Mana: &f{value}"
total-max-stamina: "&e Total Stamina: &f{value}"

Item Type Filtering

The item-types option controls which MMOItems item types show this stat in their GUI editor. This prevents clutter — damage stats only show on weapons, defense on armor, etc.

ValueDescription
allShow on all item types
weaponAll weapon types (sword, bow, staff, etc.)
armorAll armor types (helmet, chestplate, etc.)
equipmentArmor + accessory
handheldWeapons + tools + offhand
swordSpecific type by name
!blockExclude a specific type (prefix with !)
 
# Examples
item-types: [sword, dagger, spear]       # Only melee weapons
item-types: [armor, accessory]           # Defensive gear
item-types: [all, !block, !miscellaneous] # Everything except blocks/misc
item-types: [weapon]                     # All weapon types

Custom Formulas

For advanced users, the formula type lets you write arbitrary math expressions: 

# Effective HP = MaxHP × (1 + Armor/100 + DmgReduction/100)
effective-hp:
  type: formula
  formula: "{player.MAX_HEALTH} * (1 + {player.ARMOR}/100 + {player.DAMAGE_REDUCTION}/100)"
  decimals: 0
  show-enable: true
  item-types: [armor]
  material: TOTEM_OF_UNDYING

# Melee DPS = Total Damage × Attack Speed
melee-dps:
  type: formula
  formula: "{player.ATTACK_DAMAGE} * (1 + {player.WEAPON_DAMAGE}/100) * {player.ATTACK_SPEED}"
  decimals: 1
  item-types: [weapon]

Available Placeholders

PlaceholderDescription
{player.<STAT>}Total stat from the player's StatMap (all sources combined)
{item.<STAT>}Base stat from the item's NBT only (ignoring other equipment)

Supported operators: + - * / ^ ( )

OmniSpawner — Overview

OmniSpawner is a MythicMobs-native spawner system with stacking support for spawners, entities, and item drops. Create custom spawner blocks for any MythicMob via a GUI-driven workflow.

Features:

  • Create spawners for any registered MythicMob
  • Stack identical spawners together (x1 → x128)
  • Automatic entity stacking (merge nearby same-type mobs)
  • Automatic drop stacking (merge nearby identical items)
  • Real-time hologram with countdown timer above spawners
  • Kill-all / Kill-one modes for stacked mob deaths
  • Full GUI management system

Installation

  1. Requirements: Paper 1.21+, MythicMobs 5.x Premium.
  2. Drop OmniSpawner-1.0.jar into plugins/.
  3. Restart the server. Config generates at plugins/OmniSpawner/config.yml.

Creating Spawners

  1. Run /ospawner (or /os) to open the Spawner Manager GUI.
  2. Click the Create New Spawner button.
  3. Browse or search for the MythicMob you want. Click it.
  4. The Edit GUI opens — configure spawn rate, limit, radius, stacking.
  5. Click the Get Spawner Item button to receive the spawner.
  6. Place the spawner block in the world. A hologram appears above it.

Stacking System

Spawner Stacking

Hold the same spawner type and left-click a placed spawner to stack them. Shift+left-click adds the entire held stack at once. The hologram updates with the new count.

Entity Stacking

When enabled, nearby mobs of the same MythicMob type are merged into a single entity with a count display (e.g. "Zombie x15"). Configured in the entity-stacking section.

Kill Modes

ModeBehavior
kill-one Each kill reduces stack by 1, drops loot once. Stack: 100→99→98...
kill-all One kill destroys entire stack, drops loot × stack count.

Configuration

# config.yml — Key settings

spawner-stacking:
  enabled: true
  max-stack: 128

entity-stacking:
  enabled: true
  max-stack: 256
  kill-mode: "kill-one"

spawn-mode: "multiply"       # multiply or single

drop-stacking:
  enabled: true
  max-stack: 1024

hologram:
  enabled: true
  format:
    - "7FF55{name}"
    - "򇨣x&#FFFFFF{stack} 򇨣| &#FFAA00 {countdown}"

Spawner Drops from Mobs

When a MythicMob is killed, if a spawner definition exists for that mob type, there is a configurable chance to drop the spawner item. Only mobs that already have a spawner definition will ever drop spawners — no random drops from undefined mobs.

How it works

  • The plugin checks if the killed mob's internal ID matches any spawner definition
  • If a match is found, it rolls the drop chance (default: 0.1%)
  • If successful, the spawner item drops at the mob's death location with a glow effect
  • The killer receives a configurable notification message

Multiple Spawner Definitions

A single mob type can have multiple spawner definitions (e.g. "zombie-farm" and "zombie-grinder"). When that mob dies, each spawner definition rolls its drop chance independently, so there are no conflicts.

Config

# config.yml
spawner-drop:
  enabled: true
  default-chance: 0.1               # 0.1% default drop chance
  drop-message: "7FF55A rare spawner dropped!"

Per-Spawner Override

You can override the drop chance for a specific spawner in spawners.yml. If not set, the global default-chance is used. 

# spawners.yml (auto-generated)
spawners:
  skeleton-king:
    mob-id: SkeletonKing
    display-name: "&#FF5555Skeleton King Spawner"
    spawn-rate: 600
    drop-chance: 0.5             # 0.5% for this spawner specifically

  skeleton-basic:
    mob-id: SkeletonKing         # Same mob, different spawner — no conflict
    display-name: "&#FFAA00Basic Skeleton Spawner"
    spawn-rate: 200
    # No drop-chance set → uses global 0.1%
Drop chance values The drop-chance value is a percentage. Common values: 0.1 = 0.1% (1 in 1000), 0.5 = 0.5% (1 in 200), 1.0 = 1% (1 in 100), 5.0 = 5% (1 in 20).
